RSA АЛГОРИТМІ БОЙЫНША ТАПСЫРМАЛАРДЫ

CRYPTOOL 2 БАҒДАРЛАМАСЫНДА ОРЫНДАУ

 

 

CrypTool 2 — бұл криптографияны зерттеу мен оқытуға арналған ашық бастапқы кодты бағдарлама. Ол шифрлау, дешифрлау және криптоанализ алгоритмдерін визуалды түрде зерттеуге мүмкіндік береді. CrypTool 2 классикалық және заманауи криптография әдістерін, хэш-функцияларын, сондай-ақ ашық кілтті шифрлау алгоритмдерін қолдайды. Бағдарлама оқу процесінде қолдануға өте ыңғайлы, себебі оның интерфейсі қарапайым, ал алгоритмдердің орындалуын блок-схема түрінде көруге болады. CrypTool 2 — криптографияның теориялық негіздерін тәжірибемен ұштастыру үшін таптырмас құрал.

Text Input (Мәтінді енгізу) – Бұл блок арқылы мәтінді енгізуге болады. Оны шифрлау, дешифрлау немесе басқа криптографиялық операциялар үшін бастапқы дерек ретінде қолданады.

Text Output (Мәтінді шығару) - Бұл блок операция нәтижесін (шифрланған мәтін, дешифрланған мәтін және т.б.) көрсету үшін қолданылады.

String Encoder (Жолды кодтау) - Мәтінді белгілі бір кодтау форматына түрлендіреді (мысалы, ASCII, UTF-8 және т.б.). Бұл блок криптографиялық операцияларды орындау үшін деректерді сандық форматқа ауыстырады.

String Decoder (Жолды декодтау) - Кодталған сандық деректерді бастапқы мәтінге қайта түрлендіреді. Дешифрланған мәліметтерді түсінікті мәтін ретінде көрсетуге мүмкіндік береді.

RSA Key Generator (RSA кілтін генерациялау) - RSA алгоритмі үшін ашық және жабық кілттерді (e, d, n) автоматты түрде есептеп береді. Бұл блокта бастапқы параметрлерді (p, q, e) орнатуға болады.

RSA - RSA алгоритмін іске асыратын негізгі блок. Ол ашық кілтпен мәтінді шифрлау (C= mod  n) және жабық кілтпен дешифрлау (M = mod  n) операцияларын орындайды.

1. String Decoder

Мақсаты: Кодталған сандық немесе басқа форматтағы деректерді бастапқы жол түріне қайта түрлендіреді.

Конфигурация:

·  Input format (Кіріс форматы):

·                      Бұл опция кодталған деректердің форматын (мысалы, Decimal – ондық, Hexadecimal – он алтылық) анықтайды.

·  Use separators for grouping (Топтастыру үшін бөлгіштерді қолдану):

·                      Егер деректер бірнеше бөлікке бөлінсе (мысалы, үтір немесе бос орын арқылы), бұл параметрді қосуға болады.

·  Set of separators (Бөлгіштердің жиынтығы):

·                      Бұл өріске деректердегі бөліктерді ажырататын таңбаларды енгізуге болады (мысалы, үтір, бос орын).

 

2. String Encoder

Мақсаты: Бастапқы мәтінді белгілі бір форматта (мысалы, сандық немесе символдық код) кодтайды.

Конфигурация:

·  Presentation format (Шығыс форматы):

·                      Кодталған деректердің қандай форматта ұсынылатынын анықтайды (мысалы, Decimal – ондық, Binary – екілік, Hexadecimal – он алтылық).

·  Maximum length (Максималды ұзындық):

·                      Кодталған жолдың максималды ұзындығын шектейді. Мысалы, ұзақ жолдарды шифрлау кезінде деректердің өлшемін реттеуге болады.

 

3.RSA

Конфигурация сипаттамасы:

1.                 Action (Әрекет):

·                      Бұл параметр арқылы блоктың негізгі функциясын таңдай аласыз:

·                      Decryption (Дешифрлау): Шифрланған деректерді бастапқы деректерге қайта түрлендіру.

·                      Егер шифрлау қажет болса, "Encryption" параметрі таңдалуы мүмкін.

2.                 Cores used (Қолданылатын ядролар):

·                      Бағдарлама көп ядролы процессорларды қолдана алады.

·                      Тек бір процессор ядросын пайдалану. Көп ядролы режимде жылдамдықты арттыру үшін мәнін өзгертуге болады (мысалы, 2, 4 және т.б.).

Block sizes (Блок өлшемдері):

1.                 Override implicit block sizes (Әдепкі блок өлшемдерін өзгерту):

·                      Бұл параметр қосулы болса, пайдаланушы енгізу және шығу блоктарының өлшемдерін қолмен өзгерте алады.

2.                 Input block size (Кіріс блогының өлшемі):

·                      Кіріс деректерін өңдеу үшін қолданылатын деректер блогының өлшемі.

·                      Бұл мәнді криптографиялық алгоритмге сәйкес реттеуге болады (мысалы, 8 байт = 64 бит).

3.                 Output block size (Шығыс блогының өлшемі):

·                      Шығарылатын деректердің блогы үшін өлшем.

·                      Ол кіріс өлшеміне сәйкес болуы мүмкін немесе алгоритмге байланысты өзгеруі мүмкін.

 

5.                 Text Input (Мәтінді енгізу) блогына M-нің мәнін жазып жобаны орындауға жібереміз.Орындауға жіберу үшін Play батысмасын басып Text Output (Мәтінді шығару) блогында С-дің мәнін көреміз.
